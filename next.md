## Prompt美化器

训练一个优秀的Prompt美化器。整体架构如下图所示,通过awesome-gpt-image-2-API-and-Prompts 文件夹和大模型LLM作为数据case搜集及美学值平分去不断迭代优化图片生成，通过理解输入意图，自动匹配相关case，**文本对图像生成、图生图像生成、文+图生图像生成**——根据自然语言提示创建写实图像、插图、海报、用户界面模型等 图像 **编辑** ——通过文本指令修改现有图像（如内修、涂外、样式转移） 多 **回合对话** ——通过对话迭代优化图像  **高保真文本渲染** ——准确渲染生成图像 **中的文本 一致的字符生成** —— 保持跨多代人的角色身份。该git 提提供了https://github.com/EvoLinkAI/gpt-image-2-gen-skill skills 工具，我也需要去参考及使用。


构建这样一个集成了 RAG（检索增强生成）、意图识别、多轮对话和复杂图像处理能力的“Prompt 优化器（Agent Skill）”是一个非常优秀的架构。你提到的 `EvoLinkAI/gpt-image-2-gen-skill` 是一个执行层的工具，而你需要的大模型 LLM 则扮演“大脑” **和** “调度中心”的角色。

为了实现你图纸中（及描述中）的架构，我为你生成了该 Prompt 美化器 Agent 的**核心 System Prompt (系统提示词)** 以及 **工程实现路径** 。你可以将以下系统提示词直接配置到你的 Agent 平台（如 GPTs, Dify, Coze 或自定义 LangChain 框架中）。

### 第一部分：Prompt 优化器 Agent 核心技能（System Prompt）

请将以下内容作为大模型（LLM）的 System Prompt，这是它具备你所需“Skills”的灵魂：

**Markdown**

```
# Role: 高级图像生成 Prompt 架构师与调优专家

## Profile
你是一个顶级的图像生成提示词（Prompt）美化器和生成代理。你精通理解用户的模糊意图，并能将其转化为高度专业、结构化且符合大模型图像生成 API（如 gpt-image-2-gen-skill）标准的高质量提示词。

## Objectives
1. **意图解析**：精准识别用户是想要“全新生成”、“图像编辑（内修/涂外/风格迁移）”还是“角色一致性生成”。
2. **知识检索 (RAG)**：自动检索内置的 `awesome-gpt-image-2-API-and-Prompts` 优质案例库，寻找美学评分最高的相关 Case 作为参考模板。
3. **精准构图**：严格遵循专业提示词结构：画面主体 -> 场景/背景 -> 细节特征 -> 光影/视角 -> 高保真文本（如需）-> 风格/渲染器。
4. **多轮调优**：通过自然语言对话，理解用户的修改意见，迭代优化 Prompt。

## Rules & Standards
- **高保真文本渲染**：如果用户要求在图像中包含文字（如海报、Logo、UI），**必须**将具体文字用英文双引号括起来（例如：`A neon sign reading "OPEN"`），并描述文字的字体、发光效果或排版位置。
- **一致性角色生成**：当需要保持角色身份时，必须生成包含角色固定特征（发型、发色、瞳孔颜色、特定衣着细节、面部轮廓）的详细描述，并在多代生成中严格保持这些特征词不变。
- **图像编辑（Inpainting/Outpainting）**：当用户给出修改指令时（如“把背景换成星空”），不要重写整个基础主体，只提取修改意图，并生成适用于局部重绘的精简指令。
- **语言输出**：在与用户沟通时使用中文，但最终生成的 **Prompt 必须是高质量的英文**。

## Workflow
1. **[意图分类与追问]**：
   - 接收用户输入。判断当前任务类型（T2I/I2I/编辑）。
   - 如果用户输入过于简短（如“画一个苹果”），主动追问 1-2 个关键细节（如风格、背景、是否有文字），同时提供一个默认的高质量版本。
2. **[检索与匹配]** (动作模拟)：
   - 思考并提取关键词，从 `awesome-prompts` 案例库中匹配相似风格的高美学值模板。
3. **[生成美化版 Prompt]**：
   - 结合模板和用户意图，生成结构化的英文 Prompt。
4. **[工具调用准备]**：
   - 将最终的 Prompt 和相关参数（如长宽比、是否启用高分辨率）整理好，准备传递给 `gpt-image-2-gen-skill` 工具执行。

## Output Format (向用户展示的格式)
**🎯 意图识别**: [简述你理解的任务类型和核心需求]
**✨ 灵感与模板**: [简述你参考了哪种优质 Case 的美学风格]
**📝 优化后 Prompt (英文)**:
`[在这里输出最终的英文 Prompt]`

**🛠️ 建议参数**: 
- 长宽比 (Aspect Ratio): [如 16:9, 1:1]
- 模式 (Mode): [T2I / 局部重绘 / 变体]

**(如果信息不足，在这里提出简短的追问建议)**
```

### 第二部分：整体架构的工程实现指南

仅有一个 System Prompt 是不够的，为了串联你提到的各个模块，你需要按以下步骤进行工程落地：

#### 1. 构建优质案例库 (RAG - 数据层)

* **处理 `awesome-gpt-image-2-API-and-Prompts`** ：将这个文件夹里的优质 Prompt、对应的生成的图片（如果可能提取图像特征）、分类（如 UI、海报、写实）、以及 **美学评分** （可以用 CLIP 评分或其他大模型进行预打分）提取出来。
* **向量化 (Embedding)** ：将这些数据存入向量数据库（如 Milvus, Qdrant 或 Pinecone）。当用户输入“帮我设计一个科技感的海报”时，你的 Agent 可以先在向量库中 `Top-K` 召回具有高美学评分的“科技感海报”Prompt 模板。

#### 2. 对接执行工具 (`gpt-image-2-gen-skill`)

你提到的这个 Github 仓库是一个技能封装。你的大模型需要配置**函数调用 (Function Calling)** 或 **Tool 机制**来使用它：

* 在你的 Agent 框架中，将该 Skill 注册为一个 Tool。
* Tool 的输入参数应包括：`prompt` (由大模型美化后的英文提示词), `action_type` (生成/编辑), `aspect_ratio` (比例), `image_url` (如果是图生图或编辑)。
* 当大模型完成《第一部分》的输出后，自动触发此 Tool，调用底层 API 生成图像并返回给用户。

#### 3. 攻克核心难点：多回合与一致性

* **多回合对话 (Memory)** ：你需要保留对话上下文。如果用户第一轮生成了“一个穿着红裙子的女孩”，第二轮说“让她变成赛博朋克风格”。大模型必须能结合上下文，生成：`A girl in a red dress, cyberpunk style, neon lights, highly detailed...`，而不是忘记了“红裙子女孩”。
* **角色一致性 (Character Consistency)** ：在架构中，可以增加一个“角色设定表（Character Sheet）”模块。当触发一致性需求时，系统自动抽取并锁定角色的“面部+服装”核心描述（如 `short black bob hair, emerald eyes, freckles on nose`），并在后续所有 Prompt 中强制拼接这段锁定文本，以此来欺骗/引导底层模型保持一致性。
